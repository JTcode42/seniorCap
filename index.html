The Problem

Traditional data networks lack diversification and security

It is convention for information to be stored all together by a centralized entity such as a company or government. As a result, one must put faith in these centralized agencies to secure their network, devices, and the respective information.

This practice works fine until these centralized entities tamper with the data, begin misusing information, are compromised by rogue employees, or are most commonly hacked due to a single point of failure.

The latter of which is frequently observed among IoT devices.

A Potential Solution

A special database relying on cryptography to ensure data integrity

Think of a database that has multiple copies of itself. All of which automatically update in unison. If one copy gets tampered with then all of the other copies could highlight this discrepancy so that the database could be resynchronized—and therefore resecured—before continuing. This way it would prevent a bad actor from accessing the database and would allow the user to be aware of any attacks.




Blockchain Basics

Blockchain technology is essentially a database/ledger that has copies of itself stored on multiple devices (ie, servers, computers, IoT devices) within a specific network. Each of these devices is a node.

The nodes hosting the ledger are distributed to varying extents so that they exist independently of one another. In order for updates to occur to the ledger, the majority of these nodes must verify the legitimacy of the proposed changes.

Upon verification, the new data is packaged into a block that is crpytographically encrpyted or hashed. This secure block is then chained onto the end of the existing blockchain. Note that every block is immutable.

Blockchain Security Features

256-Bit Encryption

It would theoretically take the average computer a few decades to crack the cryptography algorithm that protects the blockchain

Distributed Ledger

Centralized ledgers introduce a single point of failure since hacking a part of the network often allows you to breach other parts as well. By distributing the nodes away from a centralized entity it increases the difficulty for hijacking the network

Immutable History

Every action on the blockchain is recorded and cannot be reformed. An affordance of this immutability is that the integrity of the data is always maintained

Crpytographic Keys/Hashs

Only those equipped with a specific private key can decrypt [& encrypt] specific data. In cases where you care more about the validity of the data rather than the content of the data itself, hashing is used since it represents the authenticity of the encrypted data without having to unencrypt it

Byzantine Fault Tolerance

To modify the data, 51% of the nodes must accept the data as valid. Consequently, you would need to gain majority control of the nodes at the same time in order to falsify data

Merkle Trees

The output from one equation is the input for the next and so forth. The result is a chain of equations linked together. If any one equation is altered the entire chain is affected. This is the process behind hashing which enables immutability & creates the chain itself
